# ❓ 常见问题解答

## 1. GitHub.io 可以访问吗？

**简短回答**：可以，但需要额外配置。

### 当前状态
- ✅ **Vercel 部署**：`https://jeffrey-lee-blog.vercel.app` - 已正常工作
- ❌ **GitHub.io**：`https://jeffreylee.github.io/JeffreyLee_blog` - 还未配置

### 为什么 GitHub.io 不能直接访问？

GitHub Pages 只支持**静态网站**（HTML/CSS/JS），但你的博客是：
- Next.js 应用（需要服务器运行）
- 有数据库（PostgreSQL）
- 有 API 路由（登录、上传等）

这些功能 GitHub Pages **不支持**。

### 解决方案

#### 方案 1：只使用 Vercel（推荐）⭐
- Vercel 免费且功能完整
- 支持数据库、API、服务器端渲染
- 性能更好，全球 CDN

#### 方案 2：GitHub.io 重定向到 Vercel
如果你想保留 `username.github.io` 地址，可以设置重定向：

1. 在 GitHub 仓库设置中启用 GitHub Pages
2. 创建一个 `index.html` 文件自动跳转到 Vercel：

```html
<!DOCTYPE html>
<html>
<head>
  <meta http-equiv="refresh" content="0; url=https://jeffrey-lee-blog.vercel.app">
</head>
<body>
  <p>正在跳转到博客...</p>
</body>
</html>
```

#### 方案 3：使用自定义域名（最佳）⭐
绑定你自己的域名（如 `blog.yourname.com`）到 Vercel，这样：
- 更专业
- 更容易记忆
- 可以同时配置多个子域名

## 2. 登录不需要密码吗？

**是的，不需要密码！** 使用的是**邮箱魔法链接登录**。

### 登录流程

1. **点击"登录"按钮**
2. **输入你的邮箱**（在 `ADMIN_EMAILS` 中配置的邮箱）
3. **检查邮箱**，会收到一封登录邮件
4. **点击邮件中的链接**，自动登录

### 为什么不需要密码？

- ✅ **更安全**：每次登录都需要访问邮箱，即使密码泄露也无法登录
- ✅ **更方便**：不需要记住密码
- ✅ **更现代**：很多网站都用这种方式（如 Medium、Notion）

### 如果收不到邮件？

检查以下几点：

1. **检查环境变量**：
   - `RESEND_API_KEY` 是否配置？
   - `EMAIL_FROM` 是否配置？
   - 如果没有配置，登录功能会跳过发送（开发模式）

2. **配置邮件服务**（推荐使用 Resend）：
   ```bash
   # 1. 注册 Resend：https://resend.com
   # 2. 获取 API Key
   # 3. 在 Vercel 环境变量中添加：
   RESEND_API_KEY=re_xxxxx
   EMAIL_FROM=noreply@yourdomain.com
   ```

3. **检查垃圾邮件文件夹**

## 3. 每次修改都要重新部署吗？

**是的，但不用担心！** Vercel 有**零停机部署**。

### 部署流程

1. **你推送代码到 GitHub**
2. **Vercel 自动检测到更新**
3. **Vercel 在后台构建新版本**（约 1-2 分钟）
4. **构建完成后，瞬间切换到新版本**
5. **旧版本继续服务，直到新版本就绪**

### 用户访问会怎样？

#### ✅ 部署期间（构建中）
- **用户正常访问**：继续使用旧版本
- **不会中断**：服务不停止
- **无感知**：用户感觉不到正在部署

#### ✅ 部署完成后（切换）
- **瞬间切换**：新版本上线
- **无缝体验**：用户刷新页面就能看到新内容
- **零停机**：没有服务中断

### 部署时间

- **首次部署**：2-3 分钟（安装依赖、构建、数据库迁移）
- **后续更新**：1-2 分钟（只构建变更部分）
- **用户影响**：**零影响**，旧版本继续服务

### 如何更新博客？

#### 方法 1：使用更新脚本（最简单）⭐
```bash
./update.sh "更新了文章内容"
```

#### 方法 2：手动 Git 命令
```bash
git add .
git commit -m "更新说明"
git push
```

#### 方法 3：在 Vercel 后台编辑（不推荐）
- 可以直接在 Vercel 后台编辑文件
- 但建议用 Git，方便版本管理

### 查看部署状态

1. 打开 Vercel 项目页面
2. 点击「Deployments」标签
3. 可以看到：
   - 🟡 **Building**：正在构建
   - 🟢 **Ready**：部署完成
   - 🔴 **Error**：构建失败（点击查看日志）
